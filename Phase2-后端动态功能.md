# Phase 2：后端动态功能开发

> 目标：把 Phase 1 的假数据替换为真实数据，实现随记 CRUD 和联系表单发送。

---

## 2.1 数据库搭建

- 设计随记数据表（id、正文、心情、天气、温度、位置、图片列表、创建时间、更新时间）
- 创建数据库并连接项目
- 配置数据库迁移

---

## 2.2 随记后端 API

- 创建随记 CRUD 接口（增/删/改/查）
- 新建随记时自动获取当前天气（调用天气 API，返回天气类型+温度）
- 图片上传接口（随记配图存储）
- 接口权限校验（仅管理后台可增删改，前端浏览仅可查）

---

## 2.3 随记管理后台 `/admin`

- 登录页面（密码校验）
- 随记列表（表格展示，支持编辑/删除）
- 新建随记页面：
  - 自动填入当前日期
  - 天气自动获取或手动选择（☀️/⛅/🌧️/❄️）
  - 心情 emoji 选择器
  - 位置手动输入
  - 正文编辑器（支持 Markdown）
  - 图片拖拽/粘贴上传并即时预览
- 编辑/删除已有随记

---

## 2.4 随记前端接真实数据

- 将 Phase 1 随记页面的假数据替换为从数据库读取
- 日历视图按月份查询当月有随记的日期
- 时间线视图无限滚动分页加载
- 图片灯箱接入真实图片

---

## 2.5 联系表单后端

- 联系表单提交接口
- 接入邮件发送服务，表单提交后发送邮件到博主邮箱
- 前端表单接通后端（提交 Loading、成功/失败 Toast、表单清空）

---

## 2.6 项目集接 GitHub 数据

- 接入 GitHub API，获取项目仓库的实时 Star 数和 Fork 数
- 首页 GitHub 贡献热力图接真实数据
- 首页数据卡片数字改为从真实数据计算

---

**Phase 2 完成标志**：随记可正常增删改查，联系表单可正常发邮件，GitHub 数据实时展示。
